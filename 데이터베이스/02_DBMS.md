## DBMS 등장 배경
### 파일 시스템(file system)
 - 데이터를 파일로 관리하기 위해 파일을 CRUD를 제공하는 SW
 - 응용 프로그램마다 필요한 데이터를 별도의 파일로 관리
#### 파일 시스템의 문제점
 - 같은내용의 데이터가 여러 파일에 **중복 저장**된다. -> 저장공간의 낭비는 물론 **데이터 일관성과 무결성**을 유지하기 힘들다.
 - 응용 프로그램이 데이터 파일에 종속적이다. -> **데이터 종속성**
	- 사용하는 파일의 구조를 변경하면 응용 프로그램도 함께 변경해야한다.
 - 데이터파일에 대한 동시공유, 보안, 회복 기능이 부족하다.
 - 응용 프로그램 개발이 쉽지않다.

## DBMS의 정의 및 특성
### DBMS
 - 파일시스템의 문제를 해결하기 위해 제시된 소프트웨어
 - 조직에 필요한 Data를 DB에 통합하여 저장하고 관리함
### DBMS에서의 데이터 관리
![image](https://user-images.githubusercontent.com/97611103/178095747-618c1b8f-96ac-4dde-ab60-4379c475a3d1.png)
 - 여러 응용프로그램들이 DB를 공유해서 사용한다.

### DBMS의 주요 기능
 - **정의 기능(DDL)** : DB의 구조를 정의하거나 수정할 수 있다.
 - **조작 기능(DML)** : CRUD 연산을 할 수 있다.
 - **제어 기능(DCL)** : 데이터를 항상 정확하고 안전하게 유지할 수 있다.

### DBMS의 장단점
 |장점|단점|
 |:---:|:---:|
 |데이터 **중복을 통제**할 수 있다.| **비용**이 많이든다.|
 |데이터 **독립성**이 확보된다.</br> |백업과 회복 방법이 복잡하다.|
 |데이터를 동시 **공유**할 수 있다.</br>|중앙 집중 관리로 인한 취약점이 존재한다.|
 데이터 **보안**이 향상된다.</br>
 데이터 **무결성**을 유지할 수 있다.</br>
 **표준화**할 수 있다.</br>
 장애 발생 시 회복이 가능하다.</br>
 응용 프로그램 개발 비용이 줄어든다.|</br>

## DBMS의 발전과정
### 1세대 : 네트워크 DBMS, 계층 DBMS
 - 네트워크 DBMS : 데이터베이스를 그래프 형태로 구성 ex)IDS(Integrated Data Store)
 ![image](https://user-images.githubusercontent.com/97611103/178096150-67db0fba-2000-4af0-a578-c1b3aad485aa.png)
 - 계층 DBMS : 데이터베이스를 트리 형태로 구성 ex) IMS(Information Management System)
 ![image](https://user-images.githubusercontent.com/97611103/178096162-7b0bad73-9587-4347-a26e-3a7925d6c1bc.png)

### 2세대 : 관계 DBMS
 - 관계 DBMS : 데이터베이스를 테이블 형태로 구성 ex)오라클, Access, MySQL
 ![image](https://user-images.githubusercontent.com/97611103/178096221-dc9f8f78-0a1e-4de2-b527-ec9990065017.png)

### 3세대 : 객체지향 DBMS, 객체관계 DBMS
 - 객체지향 DBMS : 객체를 이용해 DB를 구성
 - 객체관계 DBMS : 객체DBMS + 관계 DBMS
